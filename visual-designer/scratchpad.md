2026-06-10 — 3 legacy scenes (init helper) — designer found live bug: when the highlight LEFT a card, that just-left card re-animated. Cause: the keyed remount into the idle state made framer-motion replay each motion element's `initial → animate` ENTRANCE transition on mount (even though phase was already FINAL). Fix: `const init = (v) => active ? v : false` and set every `initial={init(...)}` — inactive cards get `initial={false}` so they mount straight at the final state with NO entrance animation; active cards still animate in. Test transient animation bugs deterministically (drive via hover, capture at fixed offsets), not by eyeballing the auto-loop.

2026-06-10 — hero-cards/stage-original/stage.css (.cv-play) — Creator storyboard play button sat in the bottom-right of the last frame, not centered. Cause: it centered via CSS `transform: translate(-50%,-50%)` BUT it's a `motion.span` animating `scale` — framer-motion owns the `transform` property and writes only the scale, so the translate-centering was overridden → element's top-left corner landed at center → offset by half its size. Use `inset:0; margin:auto` (definite size) or fl/grid centering on the parent instead. Same root cause family as the initial={false} fix: framer owns transform + initial.
