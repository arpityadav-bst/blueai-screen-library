2026-06-12 — agent.css (.ag-odds-note) — REMOVED the `max-width: 70ch` prose cap so the table's disclaimer footnote spans the table width (1152px), dropping it from 4 cramped lines floating in empty space to 2 aligned with the table. **Root cause + principle:** I'd styled it as PROSE (70ch readability measure) when it's a TABLE FOOTNOTE — and a caption/footnote attached to a table or figure takes the WIDTH OF THE ELEMENT IT ANNOTATES, not a prose measure, because it's reference text read in the table's context (its edges should align with the table's). Fixed the symptom before the cause. → reasonings/taste candidate: "width-by-role — a figure/table caption inherits the figure's width." (Also added `text-wrap: pretty` to .ag-disclaimer + .ag-sub for the orphan rule 14 — those keep their prose caps, they're not footnotes.)
